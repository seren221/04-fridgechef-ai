# 产品极速验证：二级域名配置与验证 SOP (V1.0)

## 核心原则 (Core Principles)
*   **速度第一**：验证期的核心是“快”。不要在基础设施上浪费时间。
*   **成本归零**：在产生收入之前，严禁投入任何域名购买成本。
*   **数据驱动**：无数据的上线是无效上线。必须通过 PV 和转化率决定生死。

## 1. 域名配置铁律
*   **禁止购买**：严禁在验证期（日收入 < $10）购买顶级域名 (.com/.net)。
*   **默认部署**：所有项目必须默认使用 Vercel 提供的免费二级域名（如 `project-name.vercel.app`）。
*   **升级条件**：仅当单日付费金额超过 $10，且持续 3 天以上时，方可申请购买顶级域名。

## 2. 伪支付验证逻辑
*   **不接真支付**：验证期严禁对接 Stripe/微信支付真实接口。
*   **Ghost Pay**：必须使用 `GhostPayButton` 组件。
*   **转化定义**：用户点击“支付”按钮并成功提交邮箱，视为一次有效“转化”。
*   **内测话术**：弹窗必须告知用户“恭喜触发内测彩蛋！输入邮箱获取早鸟 5 折特权及上线通知”。

## 3. 数据监控红线
*   **埋点必装**：上线前必须配置 GA (Google Analytics) 或其他统计工具。
*   **48小时生死线**：
    *   上线 48 小时内，若 PV < 50，立即停止推广，优化文案或换题。
    *   上线 48 小时内，若支付点击率 (CTR) < 2%，立即停止开发，寻找下一个 Top 1。

## 4. 止损机制
*   **7天原则**：如果 7 天内无任何付费转化（这里指“伪支付”转化），立即停止开发。
*   **复盘归档**：失败项目需归档至 `failed-projects` 目录，并填写失败原因分析。
